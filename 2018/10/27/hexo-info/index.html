6、指令

6.1、启动本地服务器

1)hexo s 是 hexo server 的缩写，命令效果一致；
2)预览的同时可以修改文章内容或主题代码，保存后刷新页面即可；
3)对Hexo根目录_config.yml的修改，需要重启本地服务器后才能预览效果。

6.2、新建一篇标题文章

1)因为标题中有空格，加上引号；
2)进入到source/_ posts文件夹中；

6.3、清除缓存文件。

1)清除缓存文件 db.json 和已生成的静态文件 public；
2)网站显示异常时可以执行这条命令试试；

6.4、生成网站静态文件。

1)生成public文件夹，修改配置文件后要执行；
2)hexo g 是 hexo generate 的缩写，命令效果一致；
3)生成网站静态文件到默认设置的 public 文件夹。

6.5、部署到GitHub仓库

1)hexo d 是 hexo deploy 的缩写，命令效果一致；
2)自动生成网站静态文件，并部署到设定的仓库。
